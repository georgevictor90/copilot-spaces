## Purpose
Ensure effective collaboration and clear accountability across all roles throughout the project lifecycle. Use this checklist to verify that cross-functional responsibilities are well-defined and handoffs are smooth.

---

## Project Initiation Phase

### Product Manager
- [ ] Define problem statement and success metrics
- [ ] Identify key stakeholders and their needs
- [ ] Create initial product vision and roadmap

### Project Manager
- [ ] Establish project charter and high-level timeline
- [ ] Identify cross-team dependencies and risks
- [ ] Set up communication cadence and stakeholder updates

### UX Designer
- [ ] Conduct initial user research and competitive analysis
- [ ] Create user journey maps and identify pain points
- [ ] Align design direction with product vision

---

## Planning Phase

### Product Manager
- [ ] Prioritize backlog items with clear acceptance criteria
- [ ] Review and approve user stories with the team
- [ ] Define success metrics for each major feature

### Project Manager
- [ ] Create detailed project plan with milestones
- [ ] Coordinate resource allocation across teams
- [ ] Update risk register with planning insights

### Scrum Master
- [ ] Facilitate sprint planning and backlog refinement
- [ ] Ensure team capacity is respected in commitments
- [ ] Define team working agreements and Definition of Done

### UX Designer
- [ ] Deliver wireframes and design mockups for prioritized features
- [ ] Review designs with stakeholders and incorporate feedback
- [ ] Provide design specifications and assets to developers

### QA Lead
- [ ] Review acceptance criteria for testability
- [ ] Create initial test plan and identify automation opportunities
- [ ] Define quality gates and testing timeline

### Technical Writer
- [ ] Review planned features to identify documentation needs
- [ ] Create documentation plan with deliverables and milestones
- [ ] Set up documentation framework and templates

### Developers
- [ ] Participate in estimation and technical feasibility discussions
- [ ] Identify technical risks and dependencies
- [ ] Review design specifications for implementation clarity

---

## Execution Phase

### Scrum Master
- [ ] Facilitate daily standups and track sprint progress
- [ ] Remove blockers and escalate impediments promptly
- [ ] Monitor team velocity and sprint burndown

### Developers
- [ ] Implement features according to acceptance criteria
- [ ] Participate in code reviews and maintain code quality
- [ ] Collaborate with UX Designer on implementation fidelity
- [ ] Work with QA Lead to address bugs and test failures

### UX Designer
- [ ] Review implementation for design fidelity
- [ ] Conduct usability testing on built features
- [ ] Iterate on designs based on user feedback and technical constraints

### QA Lead
- [ ] Execute test cases and automation scripts
- [ ] Report bugs with clear reproduction steps and severity
- [ ] Verify bug fixes and regression test related features
- [ ] Monitor quality metrics and report to Project Manager

### Technical Writer
- [ ] Document new features as they are developed
- [ ] Review code changes for documentation impact
- [ ] Draft and review release notes with Product Manager

### Product Manager
- [ ] Clarify requirements and acceptance criteria as questions arise
- [ ] Validate completed features against success metrics
- [ ] Adjust priorities based on learnings and feedback

### Project Manager
- [ ] Track progress against timeline and update stakeholders
- [ ] Manage scope changes and communicate impact
- [ ] Monitor and mitigate risks in the risk register

---

## Release Phase

### QA Lead
- [ ] Complete final regression testing and sign-off
- [ ] Document known issues and workarounds
- [ ] Verify all quality gates are met

### Technical Writer
- [ ] Finalize user documentation and release notes
- [ ] Ensure documentation is published and accessible
- [ ] Update version-specific references

### Product Manager
- [ ] Review and approve release readiness
- [ ] Prepare launch communications and stakeholder briefings
- [ ] Define post-launch success metrics monitoring

### Project Manager
- [ ] Coordinate release timeline with all teams
- [ ] Ensure rollback plan is documented and tested
- [ ] Update stakeholders on release schedule

### Developers
- [ ] Complete deployment checklist and runbooks
- [ ] Monitor deployment and address any issues
- [ ] Provide on-call support during release window

---

## Retrospective & Continuous Improvement

### Scrum Master
- [ ] Facilitate retrospective meeting with the team
- [ ] Document what went well and areas for improvement
- [ ] Capture and prioritize action items with owners

### All Roles
- [ ] Participate actively in retrospective
- [ ] Share honest feedback on collaboration and processes
- [ ] Commit to specific action items for improvement

### Project Manager
- [ ] Add retrospective action items to project backlog
- [ ] Track action item completion in weekly syncs
- [ ] Update process documentation based on learnings

### Product Manager
- [ ] Review product metrics and user feedback
- [ ] Identify product improvements for next iteration
- [ ] Share learnings with broader product team

---

## Escalation & Risk Management

### When to Escalate
- **Scrum Master**: Escalates team-level blockers that cannot be resolved within the team
- **QA Lead**: Escalates quality risks that threaten release timeline or product stability
- **UX Designer**: Escalates when user research reveals fundamental issues with product direction
- **Developers**: Escalate technical risks, architecture decisions, or significant delays
- **Technical Writer**: Escalates documentation gaps that impact user experience or compliance

### Escalation Path
1. Team-level discussion (Scrum Master facilitates)
2. Project Manager and Product Manager alignment
3. Product Lead or Engineering Lead (depending on nature of issue)
4. Executive Sponsor (for business-critical decisions)

### Shared Responsibilities
- **Risk Identification**: All roles contribute to identifying risks
- **Risk Ownership**: Project Manager maintains risk register, but mitigation is owned by relevant role
- **Communication**: Project Manager coordinates stakeholder updates, but all roles contribute status
- **Quality**: Everyone is responsible for quality, not just QA Lead
- **Documentation**: All roles ensure their work is documented, not just Technical Writer
